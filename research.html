Label-noise and Adversarially Robust Learning

Perturbation can be ubiquitous in the real-world data and the proper mass can actually robustify the training of machine learning algorithms. This is common in the training practice like label smoothing, dropout and data augmentation with randomness. However, when it is excessive or deliberate, the special design should be considered in the training methods to reduce their negative impact. Motivated by this belief, we developed a range of methods to provide the references on this way.

  • Label-noise learning under the class-conditional assumption [NeurIPS'18, AAAI'19, TPAMI'23]

  • Label-noise learning with the selection, correction or regularization [ICML'19, TIP'19, AAAI'21, NeurIPS'23]

  • Adversarial learning with distillation, slack or representation learning. [ICLR'22, ICLR'23, ICML'23]

Class-, Subpopulation- and Domain- Imbalance Learning

Imbalance Learning is an old topic in machine learning area, which is still lack of the solid foundation from theory to algorithm, although it seems to be studied in (at least) two decades. The reason that we re-focus this problem is that the generalization of algorithms in a broad sense has been recently drawn more attention, especially in the context of the pretraining paradigm. The underlying evaluation metric on the holistic measure of each class, each task, each domain coincidentally is similar to the fine-grained measure in imbalance learning. This motivates us to use imbalance learning to help these typical paradigms like self-supervised learning, weakly-supervised learning or generative modeling to enhance the generalization.

  • Imbalance learning with stronger model capacity [KDD'21, KDD'22]

  • Self-supervised imbalance learning [ICML'22, NeurIPS'23]

  • Class imbalance learning under implicit disparity or heterogeneity [ICLR'23, NeurIPS'23, ICLR'24, ICLR'24]

  • Balanced generative learning. [CVPR'23, ICLR'24]
